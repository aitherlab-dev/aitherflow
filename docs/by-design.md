# By Design — осознанные архитектурные решения

Этот файл фиксирует пункты аудитов, которые были рассмотрены и признаны **"так задумано"** (by design). При повторных аудитах к ним возвращаться не нужно.

---

## Безопасность

### test_hook_command выполняет произвольные shell-команды
- **Аудит:** SEC-001 (2026-03-08, критическая)
- **Решение:** By design. Пользователь сам вводит команду для тестирования хука. Это терминал — он и должен выполнять команды. Десктопное приложение работает с правами пользователя.

### test_mcp_server запускает произвольные процессы и ходит по URL
- **Аудит:** SEC-002, SEC-003 (2026-03-08, серьёзная)
- **Решение:** By design. MCP по определению запускает внешние процессы и обращается к внешним URL. Пользователь сам настраивает MCP-серверы, это его ответственность.

### shell:allow-spawn в Tauri capabilities
- **Аудит:** SEC-006 (2026-03-08, средняя)
- **Решение:** By design. Приложение управляет CLI-процессами (claude, MCP-серверы, devtools). Без spawn оно не работает — это его архитектура.

### API-ключи передаются через IPC
- **Аудит:** SEC-004 (2026-03-08, средняя)
- **Решение:** By design. Tauri IPC — внутрипроцессное взаимодействие, не сетевое. Ключи из keyring нужны фронту для голосового ввода (Deepgram WebSocket). Вектор атаки только при XSS, но XSS в изолированном WebView с CSP — крайне маловероятен.

## Код и правила

### Инлайн-SVG в CliStatsSection
- **Аудит:** RUL-003 (2026-03-08, серьёзная)
- **Решение:** By design. Правило "только Lucide React" относится к иконкам. CliStatsSection содержит SVG-график/визуализацию — Lucide для графиков не предназначен.

---

## Архитектура

### Зависимость от кумулятивного протокола CLI
- **Аудит:** BUG-008 (2026-03-08, средняя)
- **Решение:** By design. Протокол stream-json — это API Claude CLI, не наш. Мы подстраиваемся под его поведение, а не наоборот. Если протокол изменится — адаптируемся.

### Shadow-store agentStates вне Zustand
- **Аудит:** ARC-010 (2026-03-08, средняя)
- **Решение:** By design. Map на уровне модуля — осознанное решение для производительности мультиагентного переключения. Zustand-реактивность здесь не нужна, синхронизация происходит при смене активного агента.
