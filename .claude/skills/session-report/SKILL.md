# session-report

Создай отчёт о текущей сессии в папке `devlog/`.

## Формат файла

Имя: `devlog/YYYY-MM-DD_NNN.md` (дата + порядковый номер за день, начиная с 001)

## Содержимое

```markdown
Сессия: #NNN (глобальный порядковый номер, считай по количеству файлов в devlog/)
Дата: YYYY-MM-DD
Этап: N — название этапа (из ROADMAP.md)

## Задачи
- Что планировали сделать (кратко, 2-5 пунктов)

## Реализация
- Что конкретно сделали (технически, кратко, 3-7 пунктов)

## Итог
Одно-два предложения: что теперь работает, что изменилось.
```

## Правила

- Пиши кратко и технически — пользователь сам напишет посты для соцсетей
- Не лей воду, не добавляй «мотивацию» и «выводы»
- Считай номер сессии по количеству уже существующих файлов в devlog/ + 1
- Если за день несколько сессий — увеличивай номер в имени файла (001, 002...)
