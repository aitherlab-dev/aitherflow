## Серьёзные проблемы

### [SEC-001] `read_claude_md` и `save_claude_md` не проверяют путь через `validate_path_safe()`
- **Файл:** `src-tauri/src/claude_md.rs:63-84`
- **Серьёзность:** СЕРЬЁЗНАЯ
- **Описание:** Команды принимают произвольный `path` от фронтенда и читают/записывают файл без валидации. В отличие от `read_file`, `write_file`, `delete_file` и других файловых операций в `file_ops.rs`, здесь нет вызова `validate_path_safe()`.
- **Риск:** При компрометации фронтенда (XSS через WebView) атакующий может читать и записывать произвольные файлы в системе.

### [SEC-002] `add_marketplace` и `install_plugin` — path traversal через `name`/`marketplace`
- **Файл:** `src-tauri/src/plugins/commands.rs:192-431`
- **Серьёзность:** СРЕДНЯЯ
- **Описание:** Параметры `name` и `marketplace` от фронтенда используются для построения путей (`base.join("marketplaces").join(&name)`, `base.join("cache").join(&marketplace).join(&name)`) без проверки на `..` или спецсимволы. `add_marketplace` также запускает `git clone` по URL от фронтенда без валидации — вредоносный репозиторий может содержать git hooks (post-checkout).
- **Риск:** Path traversal: `name = "../../something"` позволит клонировать/записать файлы за пределами директории плагинов.

### [SEC-003] Groq API-ключ доступен в JS-контексте WebView
- **Файл:** `src-tauri/src/voice/groq.rs:7-13`
- **Серьёзность:** СРЕДНЯЯ
- **Описание:** `voice_transcribe` принимает `api_key: String` от фронтенда. Ключ передаётся из бэкенда через `load_settings`, хранится в JS-памяти, и отправляется обратно на бэкенд для каждого запроса.
- **Риск:** API-ключ доступен в JS-контексте. При XSS или компрометации WebView ключ может быть извлечён.

### [SEC-004] `file_watcher::watch_directories` не проверяет пути

### [BUG-001] Race condition при записи chat index.json
- **Файл:** `src-tauri/src/chats.rs:86-155`
- **Серьёзность:** СЕРЬЁЗНАЯ
- **Описание:** `upsert_index_entry` и `remove_index_entry` делают read-modify-write над `index.json`. Защита `chat_lock` работает для отдельного чата, но **не для самого index-файла**. Два разных чата одновременно могут прочитать один index.json, каждый добавит свою запись — вторая запись перезатрёт изменения первой. `create_chat` вообще не берёт chat_lock перед обновлением индекса.
- **Риск:** Потеря записей в индексе чатов — чат сохранён, но не появляется в списке до rebuild_index.

### [BUG-002] `ensure_agents_file` затирает agents.json при каждом запуске

- **Серьёзность:** СЕРЬЁЗНАЯ
- **Описание:** В отличие от `ensure_projects_file` (который проверяет `if path.exists()`), `ensure_agents_file` всегда записывает пустой конфиг `{ active_agent_id: null, agents: [] }`. Каждый запуск приложения сбрасывает agents.json.

### [BUG-003] Возможно завышенная стоимость в статистике (кумулятивный usage)
- **Файл:** `src-tauri/src/conductor/stats.rs:395-398`
- **Серьёзность:** СЕРЬЁЗНАЯ
- **Описание:** `input_tokens`, `output_tokens` суммируются из **каждого** assistant-события в JSONL. Если Claude CLI отдаёт **кумулятивные** usage (каждое следующее содержит сумму всех предыдущих), суммирование даёт абсурдно завышенные числа. Правильно — брать значения из последнего assistant-события или result-события.

| **Безопасность** | Хорошо | Общий уровень хороший. Главная проблема — SEC-001 (произвольное чтение/запись файлов), SEC-002 (path traversal в плагинах) |
| **Сопровождаемость** | Удовлетв. | Много дублирования (ARC-006, ARC-007, ARC-011, ARC-012, ARC-013). 4 разных fallback для HOME (ARC-005). Нет абстракций для повторяющихся паттернов |
| **Переносимость** | Хорошо | Linux-first. Hardcoded paths — единственная проблема при портировании на macOS |
